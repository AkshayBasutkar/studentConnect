Student-Proctor Participation Management System
üìã PROJECT OVERVIEW
A web platform to digitize student participation certificate submissions. Students upload certificates with proofs, proctors review and approve/reject, admins manage the entire system with analytics.

üë• USER ROLES
STUDENT

Browse events (Twitter-style feed with filters)
Submit participations (event selection, details, file upload)
Track status (dashboard with charts)

PROCTOR

Post events (rich editor, banner upload, notifications)
Review submissions (split view: student list sidebar + detail panel)
Manage students (view, add, reassign)
View event attendance/statistics

ADMIN

Full CRUD on users/events
System analytics dashboard (KPIs, charts, trends)
Generate/schedule reports (PDF/Excel)
System settings (email, storage, categories, backups)


üèóÔ∏è TECH STACK
Frontend: React/Next.js + Tailwind CSS + Shadcn/ui
Backend: Node.js/Express or Django
Database: PostgreSQL (see schema below)
Storage: AWS S3 or local with multer
Auth: JWT tokens (access + refresh)
Email: SendGrid/Nodemailer

üíæ DATABASE SCHEMA
sqlusers (id, email, password_hash, role, first_name, last_name, phone, is_active)
students (id, user_id, usn, department, year, semester, batch, proctor_id, profile_photo_url)
proctors (id, user_id, employee_id, department, designation)
events (id, title, description, category, start_date, end_date, venue, posted_by, is_pinned, is_active, banner_url)
participations (id, student_id, event_id, event_name, role, duration_days, achievement, description, status, submitted_at, reviewed_by, proctor_feedback)
participation_proofs (id, participation_id, file_name, file_url, file_type, file_size)
notifications (id, user_id, title, message, type, is_read, related_entity_id)
audit_logs (id, user_id, action, entity_type, entity_id, details, ip_address)
Relationships: User‚ÜíStudent/Proctor (1:1), Proctor‚ÜíStudents (1:N), Student‚ÜíParticipations (1:N), Event‚ÜíParticipations (1:N), Participation‚ÜíProofs (1:N)

üé® KEY UI PAGES
STUDENT
1. Events Feed

Left sidebar: Filters (category, date range, department)
Main: Twitter-style infinite scroll cards (image, title, date, venue, description preview)
Pinned events at top
Search bar, filter tabs (All/Upcoming/Past)

Step 1: Select event (dropdown with search) OR enter manually
Step 2: Details (role, dates, achievement, description)
Step 3: Upload proofs (drag-drop, max 10 files, 50MB each)
Progress indicator, validation, save draft

3. Dashboard

KPI cards: Total/Pending/Approved/Rejected
Charts: Participation trends (6 months), category breakdown
Recent activity feed (last 5 status updates)
Participations table (sortable, filterable, searchable)

PROCTOR
1. Post Event

Form: Title, category, dates, venue, description (rich text editor), banner upload
Eligibility filters (year, department)
Options: Pin to feed, send notifications
Preview before publish

2. Student Review (Split View)

Left Sidebar: Student list (name+USN), search/filter, scroll independently
Main Panel:

Top: Student profile (photo, contact, stats)
Below: Collapsible participation cards
Click card ‚Üí Full detail modal with file viewer (images in lightbox, PDFs inline, videos embedded)
Approve/Reject with feedback



3. Events Management

Collapsible event cards

4. Review Queue

Quick preview expandable cards
Bulk approve with templates

ADMIN

üîê AUTHENTICATION
Login: Email/USN + password, remember me, rate limiting (5 attempts)
Forgot Password: Email reset link (1-hour expiry)
Session: JWT access token (1 hour) + refresh token (7 days), httpOnly cookies
RBAC: Middleware checks role on all routes, 403 for unauthorized

üîî NOTIFICATIONS
In-App: Bell icon with badge, dropdown (last 10), real-time via WebSocket/polling
Email: Event posted, submission approved/rejected, password reset, weekly digests
Types: New event (students), new submission (proctors), overdue reviews, system alerts (admins)
